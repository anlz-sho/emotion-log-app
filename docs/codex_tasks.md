# Codex Tasks

## [x] Task 1:
SwiftUI + SwiftDataでプロジェクトの基本構成を作ってください。
Emotion, EmotionGroup, EmotionLog のデータモデルを作成してください。

## [x] Task 2:
今日の感情入力画面を作ってください。
感情グループごとにボタンを表示し、複数選択とメモ入力、保存ができるようにしてください。

## [x] Task 2_1:
感情ログの保存仕様を修正してください。

* 同じ日付に複数回保存した場合、新しい EmotionLog を作らない
* 今日の既存 EmotionLog がある場合は、そのログに選択した感情を追加する
* 既存の selectedEmotionIds は削除・上書きしない
* 新しく選択した感情IDだけを追加する
* 重複する感情IDは追加しない
* メモは既存メモの末尾に時刻付きで追記する
* Home画面に「今日の記録を削除」ボタンを追加する
* 今日の記録が存在する場合のみ削除ボタンを表示する
* 削除前に確認ダイアログを表示する

## [x] Task 3:
Logs画面を追加してください。

* Home, Calendar, Ranking に続く TabView の1タブとして Logs を追加する
* 保存済み感情ログを新しい順で一覧表示する
* 日時、選択感情、メモを表示する
* 縦スクロール可能にする
* カード風で読みやすいUIにする
* 既存のMVVM構成を維持する

## [ ] Task 4:
Calendar画面を実装してください。

* 月表示カレンダーを表示する

* 記録がない日は通常表示
* 今日の日付が分かるようにする
* カレンダーの日をタップできるようにする

日付タップ時：
* その日の感情ログ一覧を表示する
* 選択感情とメモを確認できる

UI方針：
* シンプル
* やさしい色
* 一目で感情量が分かる
* カレンダーらしい余白

既存のTabView構成を維持してください。
MVVM構成を維持してください。
